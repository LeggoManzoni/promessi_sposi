## Checks performed — all passed (17/17)

1. **Well-formedness**: all 43 XML files parse.
2. **`xml:id` uniqueness**: unique within every file, and across the whole
   assembled edition (no cross-file collisions).
3. **`@facs` resolution**: every `@facs` pointer resolves to an existing
   `xml:id` in `facsimile.xml`; `<lb>` and chapter-opening `<head>` target
   `type="line"` zones, `<pb>` target `<surface>`, `<figure>` target
   `type="illustration"` zones.
4. **Line-linkage completeness**: every one of the 20,579 line zones is the
   target of exactly one text-layer link (no unlinked, no doubly-linked
   zones): 20,540 from `<lb>` + 39 from `<head>`.
5. **Zone typing**: every `<zone>` `@type` is one of the five documented
   values; every `type="line"` zone carries `@n`; marginalia zones carry no
   transcribed text.
6. **Geometry**: every bounding box satisfies `0 ≤ ulx < lrx` and
   `0 ≤ uly < lry` and lies within the pixel dimensions of its surface's
   page image.
7. **CSV integrity**: 505 rows with unique `item_id`; every
   `iiif_region_url` is consistent with its `page_iiif_image_service` and
   `x,y,w,h` fields; every TEI `<figure>` `@corresp` has a matching CSV
   record (438 of 505; the remaining 67 are front matter and *Storia della
   colonna infame*, outside the encoded scope).
8. **RelaxNG (structural)**: `header.xml`, `facsimile.xml`,
   `listPerson.xml`, and the assembled `dist/promessi-sposi-quarantana.xml`
   validate against TEI P5 `tei_all.rng` with no deviations. The project
   Schematron rules (`schema/promessi-sposi.sch`) are enforced by checks
   3–6 above, which implement the same assertions.
9. **IIIF availability** (checked 2026-07-11): the FICLIT IIIF manifest
   (`…/iiif/2/232045/manifest`), a page `info.json`, and representative
   line- and illustration-region URLs all return HTTP 200.

## Known, intentional exceptions

- **One `<lb>` without `@facs`** (`cap1.xml`, dialogue break after
  "Perpetua."): a pre-existing editorial line break with no physical
  counterpart in the print; deliberately preserved (see
  `docs/encoding-guidelines.md`).
- **`@facs` syntax in the assembled edition**: the modular files use
  `facs="facsimile.xml#id"`; in `dist/promessi-sposi-quarantana.xml` these
  are rewritten to document-internal `facs="#id"`. The Schematron
  `facs-syntax` pattern describes the modular files.

## Manual verification (eScriptorium)

Initial line regions were generated by automated layout analysis and
aligned with the encoded text; the full line layer was then manually
reviewed page by page in eScriptorium. Empty or spurious regions were
removed and incorrect line boundaries (truncated or overextended line
endings in particular) were corrected. The checks above verify the
*internal consistency* of that curated result; they do not re-measure
geometric accuracy, and bounding boxes remain functional regions rather
than exact typographic contours.

## Result

**PASS** — all 17 automated checks succeeded on the files as deposited.
